| Version | Supported |

| Latest  | ✅ Yes    |
| Older   | ❌ No     |

---

## 🛡️ Reporting a Vulnerability

We take security seriously. If you discover a vulnerability in Blood Donor BD, please report it responsibly.

### How to Report

1. **Do NOT** create a public GitHub issue for security vulnerabilities.
2. Email us directly at: **blooddonorbd025@gmail.com**
3. Include the following details:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fix (if known)

### What to Expect

| Response Time | Action |
|---------------|--------|
| Within 24 hours | Acknowledgment of your report |
| Within 3-5 days | Investigation and confirmation |
| Within 7 days | Fix or mitigation plan |
| Within 14 days | Patch release (if applicable) |

---

## 🩸 Responsible Disclosure

We kindly ask that you:

- ✅ Give us reasonable time to fix the issue before disclosing it publicly
- ✅ Work with us to understand and resolve the issue
- ✅ Do not exploit the vulnerability for any malicious purpose

---

## 🏥 Scope

This policy applies to:

- Firebase backend integrations
- User data and authentication systems
- APIs and data storage
